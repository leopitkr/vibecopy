## Phase 1: 긴급 수정 (P0) — 출시 전 필수

> 목표: "첫 사용에서 실망하지 않는 최소 품질" 확보

### 1-1. max_tokens 900 → 2000 증가

- **파일**: `lib/openai/client.ts:5`
- **변경**: `MAX_OUTPUT_TOKENS = 900` → `MAX_OUTPUT_TOKENS = 2000`
- **이유**: 900토큰으로 32개 카피 = 개당 ~28토큰(14단어). 후반부 숏폼 스크립트 body가 1~2문장으로 잘림. 5/5 페르소나에서 품질 불만
- **비용 영향**: gpt-4o-mini 기준 output $0.60/1M tokens. 900→2000은 생성당 약 $0.0007 추가. 무시 가능
- **작업량**: 5분

### 1-2. URL 입력 기능 비활성화

- **파일**: `components/GeneratePageClient.tsx`
- **변경**: URL 입력 탭 제거 또는 "준비 중 (Coming Soon)" 표시
- **이유**: URL을 넣으면 URL 문자열이 그대로 프롬프트에 전달됨. 쿠팡/스마트스토어 URL 입력 시 쓰레기 결과 → 첫 사용 신뢰 붕괴
- **방법**: SourceType 토글에서 "url" 선택 시 disabled 상태 + 안내 메시지
- **작업량**: 30분

### 1-3. 채널별 출력 구조 분기 (핵심)

- **파일**: `lib/prompts/generateCopy.ts` (프롬프트), `app/api/generate/route.ts` (응답 스키마), `components/CopyPackageView.tsx` (렌더링)
- **변경**: 채널별로 출력 카테고리와 개수를 다르게 설계

**채널별 출력 구조안:**

```
smartstore (스마트스토어):
  - 상품명 후보 5개 (네이버 검색 최적화, 50~80자)
  - 핵심 베네핏 5개 (상세페이지용)
  - 구매 포인트 5개 (구매 결정 유도)
  - 검색 키워드 제안 5개
  → 총 20개, 스마트스토어에 실제로 필요한 것만

coupang (쿠팡):
  - 상품 타이틀 5개 (브랜드+제품명+용량+특징 형식)
  - 핵심 불릿포인트 5개 (5줄 요약)
  - 구매 이유 5개
  → 총 15개, 쿠팡 상품 등록에 바로 활용 가능

social (SNS):
  - 피드 캡션 5개 (이모지 포함)
  - 스토리 문구 5개
  - 댓글 유도 5개
  - DM 멘트 5개
  → 총 20개, 인스타그램 운영에 최적화

shortform (숏폼):
  - 후킹 라인 10개 (3초 이내)
  - 풀 스크립트 5개 (hook + body + CTA, 15~30초 분량)
  → 총 15개, 숏폼 크리에이터에게 실질적 도움

affiliate (제휴/블로그):
  - 블로그 제목 5개
  - 본문 도입부 5개
  - 추천 문구 5개
  - 마무리 CTA 5개
  → 총 20개
```

**구현 단계:**
1. `lib/prompts/generateCopy.ts`에 채널별 출력 스키마 정의 + 프롬프트 분기
2. `app/api/generate/route.ts`에서 채널별 Zod 스키마 분기
3. `components/CopyPackageView.tsx`에서 채널별 렌더링 분기
4. 기존 데모(랜딩 페이지 올리브오일 예시) 업데이트
- **작업량**: 1~2일

### 1-4. 전체복사 플레인텍스트 옵션

- **파일**: `components/CopyPackageView.tsx`
- **변경**: "전체 복사" 버튼 옆에 "텍스트로 복사" 추가. 마크다운 헤더(##) 제거하고 번호+줄바꿈만
- **이유**: 스마트스토어/쿠팡 에디터에 마크다운 붙여넣으면 태그가 그대로 보임
- **작업량**: 30분

## Phase 2: 전환 구조 개선 (P1) — 출시 직후 1주 내

> 목표: "무료 사용자가 결제할 이유" 만들기

### 2-1. 무료 플랜 축소 + 품질 차별화

- **현재**: Free 3회/일 → 5/5 페르소나 "충분한데?"
- **변경안**:

| 플랜 | 횟수 | 모델 | 출력 |
|------|------|------|------|
| Guest (비회원) | 1회/일 | gpt-4o-mini | 전체 |
| Free (가입) | 1회/일 | gpt-4o-mini | 전체 |
| Standard 19,000원 | 100회/월 | **gpt-4o** | 전체 + 히스토리 |
| Pro 49,000원 | 무제한 | **gpt-4o** | 전체 + 브랜드보이스 + CSV + A/B |

- **핵심 변경**:
  - Free 3회→1회로 축소 (체험은 가능하되 업무용으로는 부족하게)
  - Standard 500회→100회로 축소 (개인 셀러 실사용량은 월 20~50회)
  - **유료 = gpt-4o** 모델 차별화 → 무료 대비 확실한 품질 차이 체감
  - 첫 가입 시 7일간 Standard 체험 제공 (결제 정보 없이)

- **파일**: `lib/constants/limits.ts`, `lib/openai/client.ts`, `app/api/generate/route.ts`, `app/pricing/page.tsx`
- **비용 계산**: gpt-4o input $2.50/1M, output $10/1M. 생성 1회당 ~$0.02. Standard 100회 = 월 $2. 충분히 수익성 있음
- **작업량**: 1일

### 2-2. 소셜 로그인 추가 (카카오 + 네이버)

- **현재**: 이메일+비밀번호만
- **변경**: Supabase Auth에 Kakao OAuth + Naver OAuth 추가
- **이유**: 한국 셀러 타겟에서 소셜 로그인 없으면 가입 전환율 50% 이상 손실
- **구현**:
  1. Supabase Dashboard → Authentication → Providers → Kakao/Naver 활성화
  2. 카카오 개발자 콘솔에서 앱 등록 + Redirect URI 설정
  3. 네이버 개발자 센터에서 앱 등록
  4. `app/login/page.tsx`, `app/signup/page.tsx`에 소셜 로그인 버튼 추가
  5. 소셜 로그인 후 `users` 테이블 auto-create 확인 (이미 트리거 있는지 확인)
- **작업량**: 1일

### 2-3. 첫 7일 무료 체험 (Standard 동등)

- **변경**: 신규 가입자에게 7일간 gpt-4o 모델 + 일 5회 제공
- **구현**: `users` 테이블에 `trial_ends_at` 컬럼 추가. 생성 API에서 trial 기간 체크
- **이유**: 무료에서 바로 유료 전환은 어려움. 품질 차이를 체험시킨 후 전환 유도
- **작업량**: 반나절

## Phase 3: 가치 강화 (P1~P2) — 출시 후 2~4주

> 목표: "ChatGPT보다 확실히 나은 이유" 확립

### 3-1. SNS/공구 채널 이모지 자동 삽입

- **파일**: `lib/prompts/generateCopy.ts`
- **변경**: social/shortform 채널 프롬프트에 이모지 규칙 추가
  - "각 줄에 맥락에 맞는 이모지 1~2개 포함"
  - 공구특화 바이브에서 "🔥", "⏰", "💕" 등 실제 공구글 이모지 패턴 반영
- **작업량**: 1시간

### 3-2. 결과 부분 재생성

- **변경**: 생성 결과에서 카테고리별 "이 섹션만 다시 생성" 버튼
- **이유**: 헤드라인은 좋은데 베네핏만 다시 뽑고 싶을 때 전체 크레딧 소모 방지
- **구현**: API에 `regenerate_section` 파라미터 추가. 해당 섹션만 프롬프트 재호출
- **작업량**: 1일

### 3-3. URL 크롤링 구현 (Phase 1에서 비활성화한 것 복구)

- **변경**: 스마트스토어/쿠팡 URL 입력 시 실제 페이지 크롤링 후 상품 정보 추출
- **구현**:
  1. `lib/scraper/` 모듈 신설
  2. cheerio 또는 puppeteer로 상품명, 가격, 상세설명, 이미지 alt 텍스트 추출
  3. 추출된 텍스트를 기존 generate 파이프라인에 전달
  4. 크롤링 실패 시 graceful fallback (텍스트 입력 유도)
- **주의**: 쿠팡/네이버 robots.txt 및 이용약관 확인 필요
- **작업량**: 2~3일

### 3-4. 한국 광고법 금칙어 필터

- **변경**: 생성 결과에서 식약처/공정위 금칙어 자동 감지 + 경고
- **예시**: "최고", "완벽", "100% 효과" 등 과대광고 표현 하이라이트
- **이유**: 셀러 신뢰 확보 + 법적 리스크 경감 → ChatGPT에 없는 차별점
- **구현**: 금칙어 사전 + 생성 후 필터링 레이어
- **작업량**: 1일

## Phase 4: 성장 기능 (P2) — 출시 후 1~2개월

> 목표: 리텐션 + 입소문

### 4-1. 브랜드 보이스 저장 (Pro 기능 실제 구현)

- 현재 pricing 페이지에 "브랜드 보이스 저장"이 Pro 기능으로 표시되어 있으나 미구현
- 사용자의 과거 생성 결과에서 톤/스타일 패턴 추출 → 프로필로 저장 → 이후 생성에 반영
- 공구 운영자(P3)에게 킬러 피처

### 4-2. CSV 다운로드 (Pro 기능 실제 구현)

- 대량 SKU 운영 셀러(P2)를 위한 엑셀 호환 다운로드
- 현재 pricing 페이지에만 표시, 미구현

### 4-3. A/B 카피 변형 (Pro 기능 실제 구현)

- 같은 상품에 대해 2가지 톤의 카피를 동시 생성
- pricing 페이지에만 표시, 미구현

### 4-4. 쿠팡/스마트스토어 SEO 키워드 추천

- 채널별 인기 검색 키워드 제안
- 네이버 데이터랩 / 쿠팡 자동완성 API 연동 검토

## 리스크 & 의사결정 필요 사항

1. **gpt-4o 비용**: Standard 100회/월 기준 월 $2. 충분히 감당 가능하나, 트래픽 증가 시 재검토
2. **쿠팡/네이버 크롤링**: robots.txt 위반 가능성. 법적 검토 후 진행 여부 결정
3. **무료 1회/일 축소**: 기존 가입자 반발 가능. 기존 사용자 3회 유지 vs 신규만 1회 정책 검토
4. **Pro 기능 미구현**: pricing 페이지에 이미 표시된 기능(브랜드보이스, CSV, A/B)이 미구현 상태. 결제 후 실망 → 환불 리스크. Phase 4 전까지 Pro 플랜 숨기기 또는 "Coming Soon" 표시 권장
