VibeCopy 합성 사용자 테스트 리포트

  ---
  1. Executive Summary

  한 줄 결론: VibeCopy는 "채널별 32개 카피를 한 번에"라는 명확한 가치를
  가졌지만, gpt-4o-mini의 900토큰 제한으로 인한 결과 품질 불안정, 소셜 로그인
  부재로 인한 가입 마찰, "ChatGPT에 프롬프트 한 번 넣으면 되지"라는 반론을
  뒤집을 결정적 한 방이 부족하다.

  지금 팔아도 되는가? → 제한된 베타 판매 가능 (아래 P0 수정 후)

  가장 큰 병목 3개:

  Persona 1: 김수진 (32세) — 초보 스마트스토어 셀러

  ┌────────────────┬───────────────────────────────────────────────────────┐
  │      항목      │                         내용                          │
  ├────────────────┼───────────────────────────────────────────────────────┤
  │ 배경           │ 육아맘, 6개월 전 스마트스토어 개설. 영양제·생활용품   │
  │                │ 위탁판매. 월매출 80만원                               │
  ├────────────────┼───────────────────────────────────────────────────────┤
  │ 현재 업무 방식 │ 도매사이트 상세페이지 이미지 그대로 복붙. 제목은 다른 │
  │                │  셀러 참고해서 직접 작성                              │
  ├────────────────┼───────────────────────────────────────────────────────┤
  │ 가장 귀찮은    │ "상세페이지 텍스트를 뭐라고 써야 할지 모르겠음. 매번  │
  │ 문제           │ 1시간씩 고민"                                         │
  ├────────────────┼───────────────────────────────────────────────────────┤
  │ 기대하는 결과  │ "복붙해서 바로 올릴 수 있는 문구"                     │
  ├────────────────┼───────────────────────────────────────────────────────┤
  │ 가격 민감도    │ 높음 (월 1만원 이상은 고민)                           │
  ├────────────────┼───────────────────────────────────────────────────────┤
  │ AI 툴 숙련도   │ ChatGPT 존재는 알지만 써본 적 없음                    │
  ├────────────────┼───────────────────────────────────────────────────────┤
  │ 구매 망설이는  │ "이런 거 써봤자 별거 없을 것 같은데"                  │
  │ 이유           │                                                       │
  └────────────────┴───────────────────────────────────────────────────────┘

  테스트 흐름:

  1. 첫 인상 (랜딩):
    - "상품 정보 하나로 판매글 32개" → "32개? 엄청 많네. 근데 진짜 쓸 만한 거
  나올까?"
    - ChatGPT 비교 테이블 → "ChatGPT는 써본 적 없어서 비교가 와닿지 않음"
    - 올리브오일 데모 → "오 이런 식으로 나오는 거구나. 근데 이게 내 상품에도
  이렇게 나올까?"
    - "무료로 카피 만들어보기" 클릭 → 여기까지는 긍정적
  2. 핵심 작업:
    - 생성 페이지 도착 → 채널/바이브 선택부터 봄
    - "스마트스토어 선택. 바이브는... 신뢰형? 후기형? 뭘 골라야 하지?" → 3초
  멈춤
    - 텍스트 입력 → 도매사이트에서 상품 설명 복붙 시도
    - "상품 정보를 더 구체적으로 입력해주세요" 에러 → 키워드 부족으로 거부됨
    - "뭐야, 그냥 복붙했는데 안 되네?" → 혼란. 뭘 더 써야 하는지 모름
    - "예시로 해보기" 발견 → 클릭 → 생성 성공
    - 결과 확인: "오 헤드라인 10개나 나왔네. 근데... DM 유도 멘트? 댓글 유도?
  나는 스마트스토어인데 이게 왜 필요해?"
    - 스마트스토어 셀러에게 DM/댓글/숏폼 스크립트는 불필요한 섹션 → 32개 중
  실제로 쓸 수 있는 건 헤드라인 10개 + 베네핏 5개 = 15개
  3. 판단 포인트:
    - 기대: "내 상품 제목이랑 상세페이지 문구"
    - 실제: "SNS/숏폼용 카피가 절반 이상"
    - 혼란: 바이브 선택, 입력 거부, 불필요한 카피 섹션
    - ChatGPT 대비: 비교 불가 (안 써봄)
    - 수정 없이 쓸 수 있는지: 헤드라인 일부만. "구어체가 너무 강해서
  스마트스토어 상세페이지에 그대로 올리긴 좀..."
    - 결제 의향: 없음. "하루 3번이면 나한텐 충분한데?"

  한 줄 총평: "필요한 건 상품 제목 + 상세페이지 문구인데, 인스타/숏폼 카피가 왜
   나오는 건지 모르겠어요."

  Persona 2: 박현우 (41세) — 쿠팡 중심 실무형 셀러

  ├────────────────┼───────────────────────────────────────────────────────┤

  │ 이유           │ 알까?"                                                │
  └────────────────┴───────────────────────────────────────────────────────┘

  테스트 흐름:

  1. 첫 인상:
    - 랜딩 → "스마트스토어 위주인 것 같은데... 쿠팡도 된다고?"
    - 채널에 "쿠팡" 있는 거 확인 → 긍정적
    - "URL 입력 가능? 상품 URL 넣으면 자동으로?" → URL 입력 모드 발견
  2. 핵심 작업:
    - URL 입력 시도 → 쿠팡 상품 URL 입력
    - 문제: URL 입력 시 실제로 URL을 크롤링하는 로직이 코드에 없음. input_type:
   "url"이지만 서버에서는 inputValue를 그대로 텍스트로 AI에 넘김. URL 자체가
  프롬프트에 들어감
    - "URL 넣었는데... 결과가 이상한데? '쿠팡닷컴슬래시...'가 상품 정보로
  들어간 건가?"
    - → URL 파싱/크롤링 미구현은 심각한 UX 기만 (UI에서는 "URL 입력"을 지원하는
   것처럼 보이지만, 실제로는 URL 문자열을 그대로 프롬프트에 넣음)
    - 텍스트 모드로 전환, 상품 정보 직접 입력 → 생성 성공
    - "쿠팡" 채널 + "신뢰형" 바이브
    - 결과: "한 줄당 15단어라고 했는데... 좀 길지 않나? 그리고 쿠팡 상품명
  규칙이랑 안 맞는 거 있네"
    - 쿠팡은 상품명에 브랜드+제품명+용량+수량+특징 형식이 있는데, 이 형식에
  맞는 "상품명"은 생성 안 됨
    - DM/댓글/숏폼 → "이거 쿠팡이랑 무관한데?" → 스마트스토어 셀러와 동일한
  불만
  3. 판단 포인트:
    - 기대: "쿠팡 상품명 + 불릿포인트 5줄"
    - 실제: "후킹 헤드라인 + SNS용 카피 패키지"
    - 결정적 문제: 쿠팡 셀러가 실제로 필요한 건 "상품명 최적화"인데, 그건 생성
  안 됨
    - ChatGPT 대비: "ChatGPT한테 '쿠팡 상품명 최적화해줘'라고 하면 바로
  해주는데..."
    - 결제 의향: 조건부. "쿠팡 상품명까지 나오면 월 19,000원 낼 수 있음. 지금은
   아님"

  한 줄 총평: "URL 입력이 안 되는 건 사기 느낌이고, 쿠팡에서 진짜 필요한 건
  상품명인데 그게 안 나옴."

  테스트 흐름:

  1. 첫 인상:
    - 랜딩 → "공구특화 바이브가 있네! 이거 괜찮은데?" → 가장 긍정적 반응
    - ChatGPT 비교 테이블 → "맞아, ChatGPT로 공구 문구 뽑으면 톤이 안 맞아서
  결국 다 고쳐야 함"
    - 데모 확인 → "DM 멘트, 댓글 유도까지 나오네. 이건 내가 맨날 만드는 거잖아"
  2. 핵심 작업:
    - 채널: "SNS" + 바이브: "공구특화" → 이 조합이 이 사용자에게 가장
  자연스러움
    - 텍스트 입력: 브랜드에서 받은 상품 설명 복붙 → 생성
    - 결과: "오 DM 멘트 괜찮은데? '이거 써보신 분 계세요?' 같은 거 실제로 쓸 수
   있겠다"
    - BUT: "근데... 내 말투가 아닌 거는 어쩔 수 없네. '솔직히 처음엔'이라는
  표현은 내가 안 쓰는 건데"
    - 브랜드 보이스 저장 = Pro (49,000원) 전용. Free/Standard에서는 톤 커스텀
  불가
    - 헤드라인 10개 중 3-4개 → "이건 약간만 고치면 쓸 수 있겠다"
    - 숏폼 스크립트 2개 → "이건 진짜 좋다. 릴스 스크립트 짜는 거 맨날 고민인데"
    - 전체 복사 클릭 → "마크다운으로 나오네? 그냥 텍스트로 나오면 좋겠는데"
  3. 판단 포인트:
    - 기대: "내 톤에 맞는 공구 안내문 풀세트"
    - 실제: "괜찮은 초안. 50%는 수정 필요"
    - ChatGPT 대비: "구조가 잡혀 있으니까 확실히 빠름. ChatGPT는 '헤드라인 10개
   뽑아줘, DM 5개 뽑아줘' 따로따로 해야 하는데 이건 한 번에"
    - 결제 의향: Standard 가능성 있음. "주 2회 공구면 월 8회. 하루 3회면
  무료로도 되긴 하는데... 한 공구당 2-3번 돌리니까 모자를 수 있겠다. 근데
  19,000원은 좀 비싸고 9,900원이면 바로 결제"
    - 핵심 전환 장벽: 브랜드 보이스가 Pro(49,000원)에만 있음 → 이 사용자에게
  가장 필요한 기능이 가장 비싼 플랜에

  한 줄 총평: "32개 한 번에 나오는 건 확실히 편한데, 결국 내 톤으로 다 고쳐야
  해서 시간 절약이 반토막."

  │ gpt-4o-mini 품질 한계 (할루시네이션, 후반부 품질   │ 3/5  │ 🟠 High    │
  │ 저하)                                              │      │            │

  ├────────────────────────────────────────────────────┼──────┼────────────┤
  │ 이모지 없는 SNS 카피                               │ 2/5  │ 🟡 Medium  │
  └────────────────────────────────────────────────────┴──────┴────────────┘

  특정 페르소나에서만 나온 문제

  핵심 진단: VibeCopy의 문제는 UX가 아니라 출력 설계다. "모든 채널에 32개 동일
  구조"는 "채널별 최적화"라는 핵심 가치를 스스로 부정한다. 스마트스토어
  셀러에게 숏폼 스크립트를 주고, 쿠팡 셀러에게 DM 멘트를 주는 것은 노이즈다.

  4. 우선순위 개선안

  P0: 출시 전 반드시 수정

  P0-1. 채널별 출력 구조 차별화
  - 문제: 5개 채널 모두 동일한 6카테고리 32개 출력. 쿠팡 셀러에게 DM멘트는
  무의미
  - 왜 중요: 이것이 "ChatGPT보다 나은 이유"의 핵심인데, 실제론 차별화가 안 됨
  - 전환 영향: 🔴 전환 불가 수준. 2번 째 사용에서 이탈 유발
  - 구현 난이도: 중 (프롬프트 + 출력 스키마 분기 필요)
  - 권장 액션: 채널별 출력 구조 재설계
    - 스마트스토어: 헤드라인 5 + 상세 베네핏 5 + 검색 키워드 5 + 상세페이지
  문구 5
    - 쿠팡: 타이틀 5 + 핵심 포인트 5 + 구매 이유 5
    - SNS: 피드 캡션 5 + 스토리 문구 5 + 댓글유도 5 + DM멘트 5
    - 숏폼: 후킹라인 10 + 스크립트 5 (hook+body+CTA)
    - 공구/제휴: DM멘트 5 + 공구안내 3 + 마감문구 5 + 후기체 5

  P0-2. gpt-4o-mini → gpt-4o (또는 토큰 증가)
  - 문제: 900토큰으로 32개 카피 → 개당 ~28토큰(약 14단어). 후반부 품질 급락,
  할루시네이션
  - 왜 중요: 결과물 품질이 ChatGPT 대비 열등하면 유료 전환 제로
  - 전환 영향: 🔴 결과 불신 → 재사용 포기 → 이탈
  - 구현 난이도: 낮 (모델명 + max_tokens 변경)
  - 비용 영향: gpt-4o-mini $0.15/1M input → gpt-4o $2.50/1M input (~16배). 생성
   1회당 약 $0.003 → $0.05. Standard 500회 = 월 $25 비용
  - 권장 액션:
    - 당장: max_tokens 900 → 2000으로 증가 (gpt-4o-mini 유지해도 품질 개선)
    - 중기: 유료 플랜은 gpt-4o, 무료는 gpt-4o-mini 차별화

  P0-3. URL 입력 시 실제 크롤링 구현 또는 제거
  - 문제: UI에 "URL 입력" 탭이 있지만 서버에서 URL 크롤링 안 함. URL 문자열이
  그대로 프롬프트에 전달
  - 왜 중요: 사용자가 URL 넣고 쓰레기 결과 받으면 → 제품 신뢰 즉사
  - 전환 영향: 🔴 첫 경험 실패 → 이탈
  - 구현 난이도: 중 (cheerio/puppeteer로 크롤링) 또는 낮 (URL 탭 제거)
  - 권장 액션: 당장은 URL 탭 제거 또는 "준비 중" 표시. 이후 크롤링 구현

  P1: 출시 직후 개선

  P1-1. 소셜 로그인 추가 (카카오, 네이버)
  - 문제: 한국 셀러 타겟인데 카카오/네이버 로그인 없음
  - 왜 중요: 이메일+비번 가입은 한국 사용자에게 높은 마찰
  - 전환 영향: 🟠 가입 전환율 30~50% 개선 예상
  - 구현 난이도: 중 (Supabase OAuth provider 추가)
  - 권장 액션: Supabase에서 Kakao OAuth + Naver OAuth 설정

  P1-2. 무료 → 유료 전환 동기 재설계
  - 문제: 하루 3회면 대부분의 개인 셀러가 충분. 결제 이유 없음
  - 왜 중요: 수익 모델 자체가 무력화
  - 전환 영향: 🔴 매출 직결
  - 구현 난이도: 중 (비즈니스 모델 재설계)
  - 권장 액션 (택 1 이상):
    - A) 무료 1회/일로 축소 + 첫 7일 무제한 체험
    - B) 무료는 2카테고리만 보여주고 나머지 잠금 (프리미엄 카테고리 개념)
    - C) 무료는 basic 모델(gpt-4o-mini), 유료는 premium 모델(gpt-4o) → 품질 차
  체감
    - D) 브랜드 보이스 저장 (Pro만) → 반복 사용자에게 킬러 기능
    - 추천 조합: A + C — 무료 1회/일(gpt-4o-mini) + Standard(gpt-4o, 3회/일) +
  Pro(gpt-4o, 무제한)

  P1-3. 복사 형식 개선 (플레인텍스트 옵션)
  - 문제: "전체 복사"가 마크다운 형식 → 스마트스토어/쿠팡 에디터에서 깨짐
  - 왜 중요: 복사 즉시 사용이 핵심 가치인데 추가 편집 필요
  - 전환 영향: 🟡 사용 편의성 저하
  - 구현 난이도: 낮 (전체복사 시 plain text 옵션)
  - 권장 액션: "전체 복사" 옆에 "텍스트로 복사" 버튼 추가

  P2: 나중에 해도 됨

  P2-1. 이모지 자동 삽입 옵션 (SNS/공구 채널)
  - 문제: SNS 공구글에 이모지 필수인데 출력에 없음
  - 구현 난이도: 낮 (프롬프트에 이모지 규칙 추가)

  P2-2. 생성 결과 부분 재생성
  - 문제: 헤드라인은 좋은데 베네핏만 다시 뽑고 싶을 때 전체 재생성해야 함
  - 구현 난이도: 중

  P2-3. 쿠팡/스마트스토어 SEO 키워드 추천
  - 문제: 실무형 셀러는 검색 키워드 최적화가 핵심 니즈
  - 구현 난이도: 높 (키워드 DB 또는 추가 API 필요)

  5. 경쟁 관점 검토

  "그냥 ChatGPT 쓰면 되지"라고 느끼는 이유

  ┌────────────────────┬───────────────────────────────────────────────────┐
  │    사용자 인식     │                    구체적 근거                    │
  ├────────────────────┼───────────────────────────────────────────────────┤
  │ "결과 품질이       │ VibeCopy = gpt-4o-mini(900토큰), ChatGPT 무료 =   │
  │ 비슷하거나 떨어져" │ gpt-4o-mini(4096토큰+), ChatGPT Plus = gpt-4o.    │
  │                    │ VibeCopy가 토큰 제한 때문에 오히려 불리           │
  ├────────────────────┼───────────────────────────────────────────────────┤
  │ "ChatGPT는 내가    │ VibeCopy는 고정 6카테고리. 사용자 커스터마이징    │
  │ 원하는 형태로      │ 불가                                              │
  │ 달라고 하면 해줘"  │                                                   │
  ├────────────────────┼───────────────────────────────────────────────────┤
  │ "무료인데 왜 돈    │ ChatGPT 무료 버전도 충분히 카피 생성 가능         │
  │ 내?"               │                                                   │
  ├────────────────────┼───────────────────────────────────────────────────┤
  │ "이미 내           │ 숙련 사용자는 자기 프롬프트를 VibeCopy보다 선호   │
  │ 프롬프트가 있어"   │                                                   │
  └────────────────────┴───────────────────────────────────────────────────┘

  뒤집으려면 강화해야 할 것

  ┌────────────┬─────────────────────────┬─────────────────────────────────┐
  │  차별화    │        현재 상태        │           필요한 상태           │
  │   포인트   │                         │                                 │
  ├────────────┼─────────────────────────┼─────────────────────────────────┤
  │ 채널       │ 프롬프트 톤만 변경,     │ 채널별 완전히 다른 출력 구조 +  │
  │ 최적화     │ 출력 구조 동일          │ 플랫폼 규정 반영                │
  ├────────────┼─────────────────────────┼─────────────────────────────────┤
  │ 즉시       │ 복사 버튼 좋음          │ + 플레인텍스트 복사 + 채널별    │
  │ 사용성     │                         │ 글자수 제한 준수                │
  ├────────────┼─────────────────────────┼─────────────────────────────────┤
  │ 프롬프트   │ ✅ 이미 달성            │ 이것만으로는 부족. 품질이       │
  │ 불필요     │                         │ ChatGPT와 동등해야 의미 있음    │
  ├────────────┼─────────────────────────┼─────────────────────────────────┤
  │ 한국 셀러  │ 공구/스마트스토어 채널  │ + 쿠팡 타이틀 최적화 + 네이버   │
  │ 특화       │ 존재                    │ SEO + 금칙어 필터링 + 광고법    │
  │            │                         │ 준수                            │
  ├────────────┼─────────────────────────┼─────────────────────────────────┤
  │ 속도       │ 5~10초                  │ ✅ 이미 ChatGPT 대비 우위       │
  ├────────────┼─────────────────────────┼─────────────────────────────────┤
  │ 결과       │ 6카테고리 분리 + 개별   │ ✅ ChatGPT 대비 명확한 우위.    │
  │ 구조화     │ 복사                    │ 이걸 더 살려야 함               │
  └────────────┴─────────────────────────┴─────────────────────────────────┘

  핵심 결론: VibeCopy가 ChatGPT를 이길 수 있는 건 **"구조화된 출력 + 즉시 복사
  + 프롬프트 불필요"**의 조합이다. 그러나 출력 구조가 채널과 맞지 않으면 이
  장점이 사라진다.

  6. 최종 판정

  제한된 베타 판매 가능

  이유:
  1. UX는 준비됨: 입력→생성→결과→복사 흐름이 깔끔하고, 로딩 UX, 에러 핸들링,
  크레딧 시스템 모두 프로덕션 수준
  2. 핵심 가치가 증명 가능: 공구 운영자(P3)에게 실제 가치 있음. "프롬프트 없이
  바로 공구 카피"
  3. 그러나 전환 구조가 깨져 있음: 무료 3회/일이면 대부분 충분 → 매출 0원 위험
  4. 채널 약속을 지키지 못함: "채널별 최적화"가 핵심 마케팅인데 실제론 동일
  출력 → 재방문 이탈

  베타 판매 조건:
  - P0-1 (채널별 출력 구조 차별화) 최소 2개 채널 (SNS/공구 + 스마트스토어)
  - P0-3 (URL 탭 제거 또는 크롤링 구현)
  - P0-2 (max_tokens 최소 2000으로 증가)

  ---
  대표가 오늘 바로 해야 할 일 5개

  - 1. lib/prompts/generateCopy.ts에서 max_tokens: 900 → 2000으로 변경 (5분.
  즉시 품질 개선)
  - 2. URL 입력 탭을 비활성화하거나 "준비 중" 표시 추가 (30분. 첫 사용 실패
  방지)
  - 3. 채널별 출력 구조 재설계 기획서 작성 시작 (2시간. P0-1이 가장 큰 전환

  - 4. 무료 플랜 3회/일 → 1회/일로 축소 검토 (비즈니스 모델 생존을 위한
  의사결정)
  - 5. 5명에게 실제 사용자 테스트 모집 (합성 테스트는 가설. 실제 셀러 피드백이
  필요)
